Ruby Classes


Our latest foray into Ruby territory leads us to classes. Classes allow you to create multiple objects that share the same or similar behaviors. Classes can, at their most basic, be thought of in terms of groupings of methods. Though they may take different input each time the class is called, the method grouping will allow for similar, or similarly-formatted outputs.

The Fonz banged on the jukebox... I don't even know how many times. If we want to know what song he caused to start playing on a given INSTANCE of a jukebox hit, we could use the above "Song" class and instance method "the_fonz" to print that information out, based on the instance variables "band" and "song". This example is about as brief as it is cheesy, but it gets the basic idea across.

Instance variables, like "band" and "song" above, are going to change relative to the instance in which they are used. We could have suggested any other band or song for our initial argument to the Song class, and whatever those would have been, would be stored in the instance variables @song and @band. We could have other instance variables like @time, @plays, or any other criteria you may associate with a jukebox, and these instance variables would all change depending on the instance of the class, and the arguments passed into them.

Again, one of the easiest ways to think of classes is to think of them as groupings of methods. More specifically, these methods would be referred to as instance methods, as they are acting within instances of a class. These instance methods can be used to store instance variable information, which changes based on the instance of a class. Some other examples of classes might include movie ticket information, with @movie_title, @running_time, or @theater_number. When it is similar sets of information that you need to store or retrieve, ruby Classes are the sorts of grouping and organization tool you will need.
