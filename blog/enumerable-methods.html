Ruby Enumerable Methods - Map


In Ruby, we sometimes deal with collections of information. Often times, we're storing these collections in containter objects like arrays and hashes. Sometimes, we want to alter the information in the existing array, sometimes we want a new array containing the altered information. The Ruby enumerable .map method is a powerful method that returns a new object containing the result of the object that .map was called upon.

.map and .each are very similar, but serve two different purposes. Whereas .each returns its receiver, having not saved anything to the object it was called upon, .map takes the output of object.map and stores it in a new container object. Check it out...

Here map has allowed us to keep our original input (the name strings) intact, while giving us a new container object with the updated values after mapping each input with the code block. This is especially useful if you need to reference original vs updated values, or you don't wish to use any methods that would permanently alter your original values.

Whereas you would typically use .each in iterative situations, and theng write the bulk of the method after passing it to an object, map handles the iteration as well as an operation as specified within the code block that follows. Depending on your purposes, this could save you a few steps, as help to keep your code short sweet and to the point.
