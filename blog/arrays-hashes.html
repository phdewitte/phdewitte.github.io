Arrays and Hashes - Container Objects Within Ruby


Recently in Dev Bootcamp's phase-0, we've been covering some basic Ruby material, including container objects like Arrays and Hashes. Both allow you to store mulitple objects or pieces of information. We'll take a look at the differences.

Arrays are the basic type of container object within Ruby. They allow you to store strings, integers, floats, and other objects within Ruby. What makes arrays particularly useful is the fact that whatever object you are storing within an array has an index number, or an nth position. This is particularly helpful if you need to select the 3rd, 83rd, or nth element within an array or arrays. The fact that every object has an index allows you to select individual or multiple elements from within the array to work with, without having to remove it from the array or treat it outside of the context of the array, unless you need to.

Hashes take containter objects a step further by allowing you to store relational objects together. Known as "Key/Value Pairs," you can store objects that relate to one another, that will stay linked within the hash. For example:



Hashes offer a little more utility in that an object may have more meaning in relation to one or more other objects within the hash. Such is the nature of the Key/Value Pair. This becomes useful in the case that you need to reference a particular piece of information in relation to to what the hash object is storing. If you had a hash storing information about a particular song, you may have keys for "Title", "Band Name", "Songwriter", etc, and corresponding values of "Revolver", "The Beatles", "McCartney, Paul".

Both arrays and hashes give you the ability to iterate through them. Essentially, this allows you to look through the contents of each, and single out a particular object, multiple objects, or all objects within the array or hash. From there, Ruby offers many internal methods pertaining to the Array and Hash classes that allow for the manipulation of however many array or hash items you have iterated over.

In the coming weeks, I expect to be able to dive into and out of arrays and hashes and select, modify, add to, or remove particular objects. Arrays and hashes seem both somewhat elementary, yet extremely powerful within the Ruby language at the same time.
